## Part 1: AO2 Definition & Core Skills

The essence of AO2: **extract information → rearrange it into patterns → perform calculations → make predictions**.

### Five Core Skills of AO2

| Skill dimension | Operational definition | Typical IGCSE task |
|:---|:---|:---|
| ① Selection & organisation | Choose and organise information from graphs, tables, text | Read a speed from a v‑t graph; identify an anomaly in a results table |
| ② Translation | Change information from one form to another (text ↔ symbol ↔ graph ↔ equation) | Draw a circuit from a description; plot an s‑t or v‑t graph from data |
| ③ Pattern recognition | Identify patterns from data or graphs and draw conclusions | Recognise direct/inverse proportion; determine acceleration from v‑t slope |
| ④ Numerical manipulation | Process data, carry out calculations, use formulae | `v = s/t` ; `F = m × a` |
| ⑤ Prediction | Make predictions or extend scientific ideas based on given information | Predict extension for a larger load from an F‑e graph |


## Part 2: Complete Experimental Data Handling Workflow

### 2.1 Identifying Variables

| Variable type | Definition | IGCSE 0625 example |
|:---|:---|:---|
| **Independent** (manipulated) | The quantity you change | Load (mass) on a spring |
| **Dependent** (responding) | The quantity you measure | Extension of the spring |
| **Control** (constant) | Quantities kept the same to ensure a fair test | Same spring, same temperature |

> Always state at least 1‑2 control variables – this is a standard requirement in Paper 4 experimental design questions.

### 2.2 Handling Anomalous Results

**Procedure**:
- Keep the anomalous value in the table (do not erase it) – circle it.
- **Exclude it when calculating the mean**.
- You may suggest “repeat the measurement” to check.

> Anomalous result: a value that clearly does not follow the general trend. It may be caused by a reading error or equipment malfunction.

### 2.3 Terminology for Data Reliability

| Term | Definition | IGCSE key point |
|:---|:---|:---|
| **Accuracy** | How close a measurement is to the true value | Instrument calibrated; systematic error minimised |
| **Precision** | How close repeated readings are to each other | Affected by random error |
| **Resolution** | Smallest change an instrument can detect | e.g. measuring cylinder: 1 mL; stopwatch: 0.01 s |
| **Random error** | Readings fluctuate up and down | Reduced by repeating and averaging |
| **Systematic error** | All readings are offset from the true value | Caused by zero error, parallax, etc. |

### 2.4 Uncertainty and Significant Figures

**Basic formulas**:
- Absolute uncertainty (single reading) = `resolution ÷ 2` (e.g. ruler with 1 mm divisions → ±0.5 mm)
- Absolute uncertainty (multiple readings) = `(max – min) / 2`
- Percentage uncertainty = `(absolute uncertainty / measured value) × 100%`

**Significant figures rules**:
- Round **only the final answer** to an appropriate number of significant figures.
- The final answer should generally have the same number of significant figures as the **least precise measurement** used in the calculation.
- Use scientific notation to avoid ambiguity.

### 2.5 Experimental Improvements – High‑score phrases

| Problem | Vague (“low‑score”) answer | Specific (“high‑score”) answer |
|:---|:---|:---|
| Low precision | “Repeat the experiment” | “Repeat three times and calculate the mean to reduce random error.” |
| Large systematic error | “The instrument might be wrong” | “Check for zero error before starting and adjust if necessary.” |
| Poor resolution | “Measure more accurately” | “Use a micrometer screw gauge (resolution 0.01 mm) instead of a ruler.” |
| Anomalous point | “Circle it” | “Identify the anomalous point, exclude it from the mean and from drawing the line of best fit, then repeat the measurement.” |


## Part 3: Complete Guide to Graph Analysis

### 3.1 Core Graph Types & Operations

| Graph | Key feature | Calculation / conclusion |
|:---|:---|:---|
| **Distance‑time (s‑t)** | Slope = speed | Horizontal → stationary; straight → constant speed |
| **Speed‑time (v‑t)** | Slope = acceleration; **area = distance** | Uniform acceleration: `a = Δv/Δt` |
| **Force‑extension (F‑e)** | Proportional region obeys Hooke’s law; slope = spring constant `k` | `F = k x` |
| **Temperature‑time** | Horizontal plateau = latent heat (change of state) | Identify melting / boiling plateau |
| **Radioactive decay (count rate vs time)** | Exponential decay; half‑life = time for count rate to halve | Read half‑life from the curve |

### 3.2 Graph Drawing Rules

| Rule | Requirement | Common mistake |
|:---|:---|:---|
| Axes | Quantity / unit; use pencil and ruler | Missing unit; wrong variable on wrong axis |
| Plotting points | Use **×** or **⊕** clearly; draw with pencil | Points too large (like a blob) |
| Scale | Spread over more than half the grid; 2 cm represents 1, 2 or 5 units | Points crammed into a corner |
| Line of best fit | A single, thin, smooth line; points evenly distributed on both sides | Dot‑to‑dot or forcing a straight line through all points |

### 3.3 Interpolation and Extrapolation

| Term | Definition | IGCSE operation |
|:---|:---|:---|
| **Interpolation** | Estimating a value **within** the data range | From the graph, draw a vertical line to the best‑fit line, then a horizontal line to read the y‑value. |
| **Extrapolation** | Predicting a value **outside** the data range | Extend the best‑fit line (straight line only) to the required x‑value, then read the y‑value. |

> ⚠️ Extrapolation carries higher uncertainty. Conclusions must be made with caution. Past papers explicitly ask to “extrapolate the line to find the time when velocity becomes zero.”


## Part 4: Command Words – What They Want

| Command | Strategy | Pitfall |
|:---|:---|:---|
| **State / Define / Give** | 1–2 short points only | Writing a long paragraph – wastes time |
| **Describe** | Say **what** happens (no reason) | Adding `because` – loses marks |
| **Explain** | Say **why** or **how**; use causal link (e.g. because…) | Only describing the phenomenon – no reason |
| **Calculate** | Show **formula → substitution (with units) → final answer** | Writing only the final number – loses marks |
| **Suggest / Predict** | Make a reasonable inference based on physics | Inventing something not supported by the context |

**Rule of thumb**: The **number of marks** = number of distinct points you should make. For a 2‑mark question, write at least two separate relevant ideas.

- Keep anomalous values in the table but exclude them from the mean.

### 5.3 Planning an Experiment (AO2 × AO3 crossover)

Step‑by‑step:
1. Set initial value of the independent variable.
2. Measure and record the dependent variable.
3. Change the independent variable (at least 4–5 different values).
4. Repeat measurements to calculate a mean.

**Control variables**: mention at least two and how they are kept constant (e.g. “use the same spring and keep the temperature constant”).

### 5.4 Error Analysis & Improvements

| Situation | Model answer phrase |
|:---|:---|
| Anomalous point | “This point is anomalous. It should be excluded from the mean and not used when drawing the line of best fit.” |
| Systematic error | “Check for zero error before the experiment” / “Use a more precise measuring instrument.” |
| Random error | “Repeat the experiment several times and calculate the mean to reduce random error.” |

### 5.5 Uncertainty & Significant Figures – Model Answers

| Calculation | Formula | Example |
|:---|:---|:---|
| Absolute uncertainty (multiple readings) | `(max – min) / 2` | Readings: 12.4, 12.6, 12.5 → mean 12.5, uncertainty (12.6−12.4)/2 = 0.1 → `12.5 ± 0.1` |
| Percentage uncertainty | `(absolute uncertainty / measured value) × 100%` | `(0.1 / 12.5) × 100% = 0.8%` |
| Significant figures | Final answer should have the same number of s.f. as the least precise measurement | `2.5 × 3.42 = 8.55 → 8.6 (2 s.f.)` |


## Part 6: Common Mistakes – Quick Check

| Common mistake | Correct practice |
|:---|:---|
| Ignoring the number of marks | Match the number of points to the mark allocation |
| Using `because` in a `Describe` question | `Describe` only states what happens – no reason |
| Writing only a number for a calculation | Show **formula → substitution (with units) → answer** |
| Confusing systematic and random error | Systematic → same direction offset; random → scatter. Remedies differ. |
| Plotting points with large blobs | Use **×** or **⊕** |
